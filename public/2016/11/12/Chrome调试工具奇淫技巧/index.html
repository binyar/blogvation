  相关的介绍网上已经有太多太多了，这里只记录一些自己开发中用到的常用技巧。

  任意输入一个网站的URL进行访问，f12调出控制台，最上面那一排工具栏有很多菜单，最要就是这些工具的使用。

elements

  从最左开始，第一个有个指针的那个小图标是选取元素的，选取元素后可以在最右边的面板改变这个元素的所有属性，面板之间还可以切换，比如查看这个元素最终的所有属性等等，第二个把当前页面用移动设备模拟访问。都是很常用的，自己体会。

console

  控制台的输入，输入之后回车就会立马执行。常用的的API有.log()/.error()/.table()/.group()等等，如下图：

sources

  资源面板，左边的目录是资源的地址目录，单击选择某个文件之后可以在右边的空白区域预览该文件，如果查看的js文件是被压缩过的，还支持格式化代码，方便直接查看

  预览js代码的时候，看到区域最左边的行号了吗？可以直接单击一下这个行号，刷新一下就可以debugger代码了。再次单击那个行号，debugger就去掉了。

  这里补充一下，如果想要改变这个js的某个逻辑，也就是修改js代码，如何才能生效呢？直接把这个js全部复制过来，然后自己在额外的一个编辑器中打开，修改你所需要修改的代码，修改完毕之后全部复制，粘贴到控制台中回车，嘿嘿，修改的代码生效了。

network

  网络面板，可以看到这个网页所有的请求，请求的耗时，请求的资源的大小，点击不同的资源类型，可以查看到这个类型的所有请求：

  XHR，这个太常用了，可以查看到发送的请求体，返回的报文信息，支持格式化查看返回参数等等

application

  应用面案，可以看到最左边的目录是一级目录Application和一级目录Storage，我们先看Application，里面有三个子目录，maniFest和Service Worker都是HTML5新引入的特性。当然因为种种原因，在实际的项目中用到的还是比较少的，毕竟动辄IE678想想都是吓死人的。这里简单的介绍一下：

maniFest

  一个存在于网站目录的文件，它的作用是当我们失去网络连接的时候同样可以进行访问到这个网站，也就是所谓的离线访问，里面的配置信息大致都是配置哪些这个网站的哪些文件是可以被离线访问到的，什么时候过期等等

worker

  Service Worker的作用是新开一个线程，独立于当前主线程，然后执行某些程序。因为我们知道js都是单线程的，如果我们让两个程序同时执行的话，就可以用到worker了。通过引用一个外部js，在这个js中通过postMessage方法传递一个参数；在引用的页面中创建一个work对象，实现onmessage事件监听之后就可接收到work传过来的数据数据然后执行程序。局限性：

  • 不能跨域加载JS
  • worker内代码不能访问DOM
  • 各个浏览器对Worker的实现不大一致，例如FF里允许worker中创建新的worker,而Chrome中就不行
  • 不是每个浏览器都支持这个新特性

  回到正题，我们的第二个一级目录是Storage，像是localStorage和sessionStorage以及cookie之类的存储信息都可以在这里查看，当然，查看了之后可不可修改呢，当然是可以的

  插上自己手机的USB数据线到电脑，然后然后手机chrome中任意打开一个网站。这时候手机连接有两点要注意：
