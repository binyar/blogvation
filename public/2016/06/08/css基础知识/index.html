盒子模型

  一个普通的块级元素从最里边到最外边由四部分组成：content（内容层）、padding（填充）、border（边框）、margin（边距），需要注意的是，当你设置了元素的宽高时，只是设置了内容层（content）的宽高，而实际的宽高还需要考虑它的填充、边框和边距。

选择器

  • 元素选择，直接写标签名称就行了：a
  • 类选择，‘.’+类名：.class-name
  • id选择，‘#’+id:#demo
  • 子元素选择，‘>’：ul>li

  当然还有更多选择，比如属性选择、伪类选择之类的就不再一一说明

权重

  如果一个元素的class定义了某个属性，然后id里面又定义了这个属性，那么最终显示的效果会是以哪个为准呢（当然实际的开发过程中肯定不可能这么简单的）？这里就涉及到了权重的问题，权重值越大的，就以它为最终显示效果：

  • 标签的权重为：1；
  • class的权重为：10；
  • id的权重为：100；

  比如p#demo1，它的权重为1+100=101；.demo2 .demo3 它的权重为10+10=20；如果demo1、demo2、demo3里面都设置了color属性，只是值不同，而p既有id=demo1又有class=”demo2 demo3”，那么它最终的color将会是demo1设置的值。

外边框合并的情形

两个或者多个毗邻的普通文档流的块级元素垂直方向的margin会出现合并。

  • 毗邻
      直白的说，两个元素是挨着的，就说是毗邻的，并且没有被非空内容、padding、border 或 clear 分隔开。也就是说，即便是两个“挨着”的元素，如果其中上一个设置了类似于border:1px solid;这样的属性的话，也是不会触发合并的。这里以两种情况来分别说明：

  我么可以清楚的看到，我们明明是设置的子元素的margin-top，最终的结果却变成了父元素margin-top:50px。如果理解了上面那句话也就很容易解释了，因为这两个元素满足合并的条件，它们的margin被合并了。

  • 普通文档流块级元素
      元素必须为块级元素，并且是处于普通文档流当中的，如果有float属性或者display:inline-block等等其他属性改变了它，就不会有合并的现象了。

  • 垂直方向
      字面意思，也就是只有margin-top和margin-bottom才会被合并，而margin-left和margin-right是不会受到影响的。

合并规则

  • 两个元素的margin都为正数的时候取较大值；
      这也就解释了上面那个例子中，为什么父元素距离顶部会有50px，如果是父元素margin-top:10px的话，最终的结果还是距离顶部50px；而如果是父元素margin-top:100px的话，结果就是距离顶部100px了。

  • 两个元素的margin都为负数的时候，取绝对值较大值；

  • 两个元素margin一正一负的时候，结果为二者相加。

防止合并

  显而易见，知道了原理，我们只需要设置某种条件来破坏他们满足合并的规则就行了，例如BFC（下文会提到），或者设置border，clear等等，方法并不统一。

BFC

  定义这种事还是交给官方来做吧，我也谈不上说什么是BFC，只能说说自己的理解：页面中凡是独立的内容区域块，就是BFC。意味着，只要这个内容区域块是独立的，它并不会受到上下文的影响，那么它就是BFC。最典型的理解就是某个元素positio:fixed。不过别的元素怎么变化，都不会影响到它的变化，它就是一个BFC。一般说来，满足以下条件，就是BFC：

  • 根元素；
  • float的值不为none；
  • overflow的值不为visible；
  • display的值不为block；
  • position的值为absolute或fixed。

  其实我们在日常的开发中经常会用到BFC，只是我们没有给它特定的取个名字而已。有了BFC，我们就可以轻松的解决一些问题了，比如上面所说道的margin合并问题，我们只需要触发一下BFC就可以轻松搞定了，再比如说，清除浮动（上文已提到），我们只需要让父元素触发BFC的条件，比如overflow:hidden，就可以轻松搞定了。
