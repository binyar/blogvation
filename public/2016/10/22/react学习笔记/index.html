import 和 require 冲突的问题

  如果一个 module 使用 export default [ModuleName]; 导出 module，而导入时却使用 require([ModuleName])，则会报错：

  解决方式是使用对应的语法：

  • import from ‘ModuleName’ 对应 export default ‘ModuleName’;
  • require (‘ModuleName’) 对应 module.exports = ‘Modulename’;。

reactTransitionGroup

  注意packge里面的名称，不是‘react-addons’也不是“react-addons-transition-group”，而是“react-addons-css-transition-group”

  reactTransitionGroup：只是在元素被添加的时候回自动在元素上面添加class（enter（刚进入），enter-active（进入完毕））属性；最后在元素被移除的时候再次添加class（leave（刚开始移除），leave-active（移除完毕））属性。通过对这两个class可以自定义样式(这些class内容是需要自己自定义的)，配合css3产生动画。

ref=“XX”无效

  如果当前的元素没有被挂载到该控件上面，而是别的地方（比如body层）；这时候通过this.refs.XXX引用是无效的，解决方法：把ref换成id，然后用jQuery选择器选择dom

最小diff

  最小different算法是用于比较内存中虚拟DOM的，如果改变了，就在页面只渲染改变的那一部分。。。这里有一个需要特别警惕的地方，如果数据量很大，比如几千几万的items，然后你在render方法里面渲染输出这些items，然而输出这些items是有一个算法的，算法返回的结果应该是一个DOM的合计，那么问题来了：每一个state的改变都会导致重新计算这些items，然后返回DOM，虽然DOM肯定不会被重新渲染，或者只渲染改变的那一部分，可是这些算法会被反反复复的执行，每一个state的改变都会去执行这些算法，这是一个非常大的内存开销，数据量稍微再大一点点可能就会直接导致页面卡顿。
